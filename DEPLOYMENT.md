# Deployment Guide

Deploy IncidentDesk for free: **Neon** (database) + **Render** (API) + **Vercel** (frontend).

Do this on your personal machine, signed into all services with your **personal** account.

---

## Order matters

The client needs the API's URL, and the API needs the client's URL (for CORS). So:

1. Database (Neon)
2. API (Render) — leave `CLIENT_URL` blank for now
3. Frontend (Vercel) — point it at the Render API URL
4. Go back and set `CLIENT_URL` on Render to the Vercel URL
5. Seed the production database

---

## 1. Database — Neon

You can reuse your existing Neon project or create a fresh one for production.

1. In the [Neon console](https://neon.tech), open (or create) your project.
2. Copy the **connection string** (`postgresql://...?sslmode=require`). You'll paste it into Render next.

## 2. API — Render

1. Push this repo to your personal GitHub (already done).
2. In the [Render dashboard](https://render.com) → **New** → **Blueprint**, connect the repo.
   Render reads `render.yaml` (repo root) and pre-fills the service.
   *(Or **New → Web Service**, set Root Directory = `server`, Build = `npm install && npx prisma generate && npm run build`, Start = `npx prisma migrate deploy && npm start`.)*
3. Set environment variables:
   - `DATABASE_URL` → your Neon connection string
   - `CLIENT_URL` → leave blank for now (fill in after step 3)
   - `JWT_SECRET` → auto-generated by the blueprint (or set your own long random string)
4. Deploy. When it's live, copy the URL, e.g. `https://incidentdesk-api.onrender.com`.
5. Verify: open `https://<your-api>.onrender.com/api/health` → should return `{"status":"ok"}`.

> **Free-tier note:** Render spins the service down after ~15 min idle, so the first request after a pause takes ~30–60s to wake. Fine for a demo — mention it in your README.

## 3. Frontend — Vercel

1. In [Vercel](https://vercel.com) → **Add New → Project**, import the repo.
2. Set **Root Directory** to `client`. Vercel auto-detects Vite (build `vite build`, output `dist`).
3. Add an environment variable:
   - `VITE_API_URL` → your Render API URL (e.g. `https://incidentdesk-api.onrender.com`)
4. Deploy. Copy the URL, e.g. `https://incidentdesk.vercel.app`.

## 4. Connect them (CORS)

1. Back in Render → your service → **Environment**, set:
   - `CLIENT_URL` → your Vercel URL (e.g. `https://incidentdesk.vercel.app`)
2. Save → Render redeploys. Now CORS allows the frontend, and cross-site cookies work (both are HTTPS).

## 5. Seed the production database

From your local machine, point the seed at the production DB once:

```bash
cd server
DATABASE_URL="<your-neon-connection-string>" npm run seed
```

This creates the demo logins (`admin@incidentdesk.dev / Admin123!`, `reporter@incidentdesk.dev / Reporter123!`) and sample incidents.

## 6. Finalize

- Open your Vercel URL and log in with a demo account — confirm the full flow works.
- Update `README.md`: replace the "coming soon" live-demo line with your real Vercel URL.

---

## Troubleshooting

- **Login doesn't persist / 401 after login** → `CLIENT_URL` on Render must exactly match the Vercel origin (no trailing slash), and the client must be loaded over HTTPS.
- **CORS error in console** → same cause: `CLIENT_URL` mismatch.
- **First request hangs ~30–60s** → Render free-tier cold start; it's awake after the first hit.
- **500 on first deploy** → check `DATABASE_URL` is set and migrations ran (`npx prisma migrate deploy` is in the start command).
